> WHAT IS PASSIVE ENUMERATION

Passive enumeration queries third-party DNS aggregation services — crtsh, hackertarget, dnsdumpster, certspotter, and others — without ever sending a single packet to the target's DNS servers.

Because all queries go to external data sources, the target sees zero traffic. No noise in their logs. No IDS alerts. No rate-limit triggers. The target cannot detect that they are being enumerated.

The tradeoff: you are limited to what these services have already indexed. Active brute-force can discover more, but at the cost of visibility.

> WHY THIS SCRIPT

Subfinder's raw output is plain text — one subdomain per line, no metadata. When you need to answer "which service found this subdomain?" or cache results across sessions, raw output falls short.

This script adds:

  • Per-subdomain source tracking — which aggregator(s) returned each result
  • Wildcard detection — *.apex entries classified and counted separately
  • SQLite 24h cache — repeat queries for the same domain serve instantly from cache.db
  • Structured JSON output — machine-readable, scriptable, pipeline-friendly

> USAGE
# basic passive query
python3 subfinder-recon.py -d target.com

# save structured JSON to file
python3 subfinder-recon.py -d target.com -o results.json

# bypass 24h cache, force fresh run
python3 subfinder-recon.py -d target.com --no-cache

# resolve IPs (adds -active -ip to subfinder)
python3 subfinder-recon.py -d target.com --active

# query only specific sources
python3 subfinder-recon.py -d target.com -s crtsh,hackertarget

# submit results to Subdomain Registry
python3 subfinder-recon.py -d target.com --submit

> SOURCE ATTRIBUTION

The script passes -oJ -cs to the subfinder binary: -oJ requests JSON output (one object per subdomain), and -cs (comma-separated sources) embeds which aggregator found each entry directly in that object.

Without --active, all "ip" fields are null and unique_ips is 0. Passing --active appends -active -ip to the subfinder command, triggering DNS resolution for every subdomain found.

Wildcards (*.apex.com) are classified separately from regular subdomains and reported in wildcard_count. They indicate a catch-all DNS record and should not be treated as distinct hosts.
